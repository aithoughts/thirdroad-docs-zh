## 基础合约

[基础合约](/solidity/base-contracts)是预先构建的智能合约，你可以在其之上构建或修改，例如[ERC721Base](/solidity/base-contracts/erc721base)合约。这些合约开箱即用，不需要实现任何功能。它们是使用[扩展](/solidity/extensions)构建的，因此可以解锁[SDKs](/sdk)和[仪表板](/dashboard)上的自定义部分中的智能功能，以便轻松与智能合约交互。

## 扩展

[扩展](/solidity/extensions)是Solididity接口，可以被仪表板识别，并在SDKs中解锁功能。由于我们的接口符合行业标准，常见的EIP也可以识别为扩展。它们是可组合的逻辑片段，可以通过[继承](/solidity/getting-started#inheritance-and-overriding-functions)轻松添加到智能合约中。

这些扩展在继承的智能合约中需要不同级别的逻辑实现\*。这个SDK中有三种不同类型的扩展:

- **扩展合约**：这些扩展是完全完整的，不需要任何逻辑来实现。通过继承这些合约，可以将功能和逻辑添加到继承的智能合约中。
- **抽象合约**：这些扩展需要实现一些逻辑。扩展上的一些功能依赖于用例，因此需要特定于你继承的智能合约的自定义逻辑。
- **接口**：这些扩展没有实现。它们通过提供函数、参数和返回类型，为编写自己的逻辑提供了指导方针。这允许你确保你的合约符合这个特定的扩展，以便你可以解锁SDKs和仪表板中的功能。

\*实现 = 使用匹配的函数签名(匹配名称、参数、可见性和返回类型)为函数编写逻辑

## 为什么使用 Solidity SDK?

- 轻松集成常见智能合约功能，包括流行的EIPs，如[ERC20](/solidity/extensions/erc20), [ERC721](/solidity/extensions/erc721), [ERC1155](/solidity/extensions/erc1155)，[权限](/solidity/extensions/permissions)，[合约元数据](/solidity/extensions/contractmetadata) 等。
- 简化开发工作流程，使你能够专注于更复杂的系统。
- 使用[基础合约](/solidity/base-contracts)为你提供了一个优化gas费和审计的基础来构建你的项目。
- 你继承的每个扩展都可以解锁更智能的[SDKs](/sdk)，自定义管理[仪表板](/dashboard)和定制的数据feeds。

例如，如果你部署了[ERC721Base](/solidity/base-contracts/erc721base)合约，它继承了包括[ERC721Mintable](/solidity/extensions/erc721mintable)在内的多个扩展，你将解锁仪表板中的“mint”按钮，并可以使用SDK中的[mint](/sdk/interacting-with-contracts/erc721mintable)功能；它会自动上传并将你的元数据固定到IPFS!
